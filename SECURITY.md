# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| v2.0.0  | :white_check_mark: Current release |
| v1.0.0  | :x: No longer supported |

We recommend always using the latest release available on the [Releases](https://github.com/CougarP/Core_Aim_pro/releases/latest) page.

## Reporting a Vulnerability

If you discover a security vulnerability in Core_Aim_pro, please report it responsibly:

1. **Do NOT open a public issue** for security vulnerabilities
2. Report the vulnerability via the official Discord server (contact the owner directly)
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 1 week
- **Fix release:** As soon as possible, depending on severity

## Scope

This policy applies to the Core_Aim_pro application distributed via GitHub Releases. It does not cover third-party dependencies or external services.

## Safe Harbor

We consider security research conducted in good faith to be authorized. We will not pursue legal action against researchers who:

- Make a good-faith effort to avoid privacy violations and data loss
- Report vulnerabilities promptly and responsibly
- Do not exploit vulnerabilities beyond what is necessary to demonstrate them

---

Thank you for helping keep Core_Aim_pro safe for everyone.
